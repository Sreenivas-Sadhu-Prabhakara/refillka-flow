restocks — approve, fulfill, stock moves

“Mark delivered” is the platform's most powerful button. One confirmed tap completes the order, moves liters out of the hub tank, and writes the impact event — DTC and institutional deliveries count their sachets right there.
Onboarding lives here too: a floating button on the Stores tab opens the new-spoke form; go-live is a single action once training is done.
PATCH /deliveries/:id → DELIVERED runs in one transaction: order + inventory + refill events + audit log.

06

Stop 06 · Under the hood

Three repos. Two services. One truth.

Sized for a small team: the fewest codebases that still scale. Every number on every screen above traces back to one PostgreSQL database through one versioned API contract.

refillka-api

NestJS monorepo

core-api (:4000) — auth, orders, deliveries, inventory, restocks, refills, partners, risk. impact-api (:4100) — SDG & challenge reporting. Shared Prisma schema + Postgres.

refillka-admin

Next.js console

Server-rendered dashboards, inline server actions, httpOnly-cookie JWT auth with silent refresh. No direct database access — APIs only.

refillka-mobile

One Flutter app

Two role-based facets, iOS + Android, Riverpod + dio with token refresh. Talks to core-api only — the field never needs the reporting service.

a refill logged at a counter in Western Bicutan → core-api → Postgres → CEO dashboard, SDG report, and the 1B counter. Same number, everywhere.
